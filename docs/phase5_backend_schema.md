# Backend Schema & Data Models Specification

---

## 1. Pydantic Data Models (Python FastAPI)

These schemas ensure absolute type-safety, auto-validation, and clear serialization on all inbound and outbound HTTP requests.

```python
from pydantic import BaseModel, HttpUrl, Field
from typing import List, Dict, Optional
from datetime import datetime
from enum import Enum

# Ingestion Job States Enum
class JobStatus(str, Enum):
    QUEUED = "queued"
    DOWNLOADING = "downloading"
    TRANSCRIBING = "transcribing"
    INDEXING = "indexing"
    GENERATING = "generating"
    DONE = "done"
    ERROR = "error"

# Represents a processing task in our pipeline
class PodcastJob(BaseModel):
    job_id: str = Field(..., description="Unique UUIDv4 identifier of the background job")
    status: JobStatus = Field(JobStatus.QUEUED, description="Active execution state")
    youtube_url: Optional[str] = Field(None, description="Source YouTube video link (null if raw upload)")
    title: Optional[str] = Field(None, description="Extracted video title or filename")
    duration: Optional[float] = Field(None, description="Podcast duration in seconds")
    error_message: Optional[str] = Field(None, description="Trace detail if status matches error")
    created_at: datetime = Field(default_factory=datetime.utcnow)

# A single timestamped snippet of transcribed speech
class Segment(BaseModel):
    start: float = Field(..., description="Segment start boundary in seconds")
    end: float = Field(..., description="Segment end boundary in seconds")
    text: str = Field(..., description="Transcribed literal text segment")
    podcast_id: str = Field(..., description="MAPPED parent job_id")

# Individual Quote Card Schema
class KeyQuote(BaseModel):
    timestamp: str = Field(..., description="Formatted timestamp (e.g. '05:12')")
    text: str = Field(..., description="Quote text segment")

# Structured Deliverables generated by LLM Summarizer
class GeneratedContent(BaseModel):
    podcast_id: str = Field(..., description="MAPPED parent job_id")
    show_notes: str = Field(..., description="Markdown compiled summary notes")
    key_quotes: List[KeyQuote] = Field(default=[], description="List of notable quotes")
    social_posts: Dict[str, str] = Field(
        default={},
        description="Keys: 'twitter', 'linkedin', 'instagram' containing platform drafts"
    )

# RAG Query Input
class QueryRequest(BaseModel):
    podcast_id: str = Field(..., description="Target database to run similarity search against")
    question: str = Field(..., description="Natural language query")

# RAG Source Citation Element
class CitationSource(BaseModel):
    start_sec: float = Field(..., description="Source start timing")
    end_sec: float = Field(..., description="Source end timing")
    timestamp_str: str = Field(..., description="Source timing formatted (e.g. '12:45')")
    text: str = Field(..., description="Source segment contextual text transcript snippet")

# RAG Query Output
class QueryResponse(BaseModel):
    answer: str = Field(..., description="Generated answer from RAG engine")
    sources: List[CitationSource] = Field(default=[], description="List of source segment citations")
```

## 2. File System Directory Structure
To avoid database maintenance and configuration for our MVP, data is persisted locally in structural directories under root directory paths:

```
[root_workspace_folder]/
├── downloads/
│   └── {podcast_id}.mp3
│       # Temporary staging location for downloaded audio streams.
│       # Deleted immediately after transcribing completes.
│
├── indexes/
│   └── {podcast_id}/
│       ├── index.faiss
│       └── index.pkl
│           # Serialization vector store database files compiled by FAISS.
│           # Loaded into memory on-demand when POST /api/query runs.
│
└── data/
    └── {podcast_id}.json
        # Complete serialized payload. Format is a unified JSON:
        # {
        #   "job_id": str,
        #   "title": str,
        #   "duration": float,
        #   "segments": [ {start, end, text} ],
        #   "content": { show_notes, key_quotes, social_posts }
        # }
```

## 3. API Routing Endpoints Signatures

*   **Endpoint:** `POST /api/process`
    *   *Input:* `Query Param / Body` containing `youtube_url` (or Multipart Form Upload if file).
    *   *Return:* `PodcastJob` (status = `queued`).
    *   *Behavior:* Validates the input URL, constructs a unique UUID `job_id`, initializes the JSON tracking cache, queues the task to `BackgroundTasks`, and returns the job detail immediately.

*   **Endpoint:** `GET /api/job/{job_id}`
    *   *Input:* Path param `job_id`.
    *   *Return:* `PodcastJob` (current status and error context).

*   **Endpoint:** `POST /api/query`
    *   *Input:* JSON body matching `QueryRequest`.
    *   *Return:* JSON body matching `QueryResponse`.
    *   *Behavior:* Checks if the requested `/indexes/{podcast_id}` directory exists, loads the index on-demand, matches similarity segments to extract the context source list, queries the LLM via LangChain QA prompt template, and yields the answer + citations.

*   **Endpoint:** `GET /api/content/{podcast_id}`
    *   *Input:* Path param `podcast_id`.
    *   *Return:* `GeneratedContent` + `title` + `segments` (merged).
    *   *Behavior:* Reads `/data/{podcast_id}.json` directly from disk and serves the payload.

## 4. Background Job State Machine & Flow

```
                     +-----------------------+
                     |        QUEUED         | (Job ID created in-memory)
                     +-----------+-----------+
                                 |
                                 v [Action: Trigger yt-dlp]
                     +-----------+-----------+
                     |      DOWNLOADING      |
                     +-----------+-----------+
                                 |
                                 v [Action: Execute Whisper (local/API)]
                     +-----------+-----------+
                     |     TRANSCRIBING      |
                     +-----------+-----------+
                                 |
                                 v [Action: Split chunks & build FAISS]
                     +-----------+-----------+
                     |       INDEXING        |
                     +-----------+-----------+
                                 |
                                 v [Action: Generate summary & social content via LLM]
                     +-----------+-----------+
                     |      GENERATING       |
                     +-----------+-----------+
                                 |
                     +-----------+-----------+
       +------------>|         DONE          | (Save to disk, delete raw audio)
       |             +-----------------------+
       |
  (Transitions)
       |
       |             +-----------------------+
       +------------>|         ERROR         | (Log error message, clear temp files)
                     +-----------------------+
```

### 4.1. Error Transitions & Cleanups
If a failure occurs at any stage:
1.  The active exception message is caught.
2.  The job's status inside `data/{podcast_id}.json` is updated to `error`, and `error_message` is populated.
3.  Any active download staging files (`/downloads/{podcast_id}.mp3`) are securely deleted to avoid disk clutter.

## 5. Security & Ingestion Safeguards
*   **API Key Management:** API tokens (`OPENAI_API_KEY`, `GROQ_API_KEY`) are read strictly from `.env` via Pydantic settings. The server aborts on startup if required configuration parameters are missing.
*   **URL Sanitation:** The YouTube URL input is sanitized using standard regex validation before passing to `yt-dlp` to prevent remote command injection attacks.
*   **Stack-Trace Leak Prevention:** In production, HTTP exceptions return structured user-facing messages (e.g. `HTTP 404: Ingestion index not found`). Raw traceback details are logged on the server but never exposed in HTTP JSON responses.
*   **CORS Settings:** The FastAPI application utilizes the standard `CORSMiddleware` configured to restrict requests to the trusted frontend origin:
    *   `allow_origins = ["http://localhost:5173"]` (or the production frontend URL).
    *   `allow_methods = ["GET", "POST"]`
